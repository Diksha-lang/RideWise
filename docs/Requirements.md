# Requirements

# Functional Requirements

The system should allow users to:

* Register Riders
* Register Drivers
* View Available Drivers
* Request a Ride
* Assign Driver to a Ride
* Complete a Ride
* Cancel a Ride
* Generate Fare Receipt

# Non-Functional Requirements

* Code should be easy to understand
* Code should be easy to maintain
* New fare strategies should be easy to add
* New driver matching strategies should be easy to add
* Low coupling between classes

# Assumptions

* Data is stored in memory
* Driver becomes unavailable after ride assignment
* Driver becomes available again after ride completion or cancellation
